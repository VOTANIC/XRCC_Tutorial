XRCC Tutorial - Solar System

Overview

You will build a simulated Solar System using XRCC in this tutorial. This tutorial builds on top of the last Museum tutorial to teach you more advanced scene building and behaviour techniques. You will learn how to build a multi-scene XR project with gravity by going through this tutorial. We recommend that you do this tutorial after finishing the Museum tutorial.

Techniques Covered​

You will learn the following techniques in this tutorial:

TechniqueSpecific Techniques Covered
Basic Scene Operation
  • Set scene skybox
  • Toggle scene gravity setting
  • Create scene
  • Duplicate scene
Advanced Scene Building
  • Using Link tool to create parent / child relationship
  • Using Offset tool to make object rotate about an external tool
Configure 3D Object
  • Set 3D object material
  • Setting physics for 3D objects
Advanced Behaviour Building
  • Using physics behaviour
  • Using variables in behaviour
  • Change scene using behaviour block
  • Using control flow behaviour blocks

Tutorial Project​

The project used for this tutorial comes in two formats: a starter project for use in this step-by-step tutorial, and a completed project for your reference after you've completed the tutorial.

Project files for the solar system tutorial can be found at:

For this tutorial, the starter project will be used to guide you build the Solar System in a step-by-step manner. All the necessary assets required for the tutorial are already included in the project, so just open the starter project in XRCC and you are ready to go with the tutorial.

Set the Scene Environment

Configure the Scene Skybox​

  1. From the Toolbar, press "Configure Tool". Click the selector next to the "Skybox (Cubemap)" option to open the Selection panel and select the "stars_milky.jpg" image in your asset library to display the image in scene. This sets the selected image as your current scene's skybox in Cubemap format.

  2. Since we are using a 360-degree image as the skybox, make sure the "Skybox 360 Image" option is selected so it's displayed correctly.

Configure Scene's Gameplay Setting​

  1. In the "Configure Tool", set the Default Tool to "Auto" and Wand Length to "50", this allows you to use a longer wand to interact with the virtual environment and teleport to different positions in the scene. Then deselect the "Use Gravity" option so user can float in space during game play.

Create the SOLAR SYSTEM

Add the Sun and Saturn Models​

  1. Open the "New Item Panel" and from the "Imported Assets" tab, select the "Sun" 3D model and use the "Spawn at World Origin" button to add the model at (0, 0, 0) position in the scene.

  1. Also from the "Imported Assets" tab, select and add the "Saturn" object to the scene as well.

  1. Use the Move transform tool to adjust the Z-Axis position of the Saturn, so that it's at the same height but far apart from the Sun, which is at the (0, 0, 0) position.

Add Other Planets using Sphere Model​

  1. We will use the Sphere 3D Model to create other planets, open the "New Item Panel" and from the "Generic Shapes" tab, select and add the "Sphere" object to the scene.

  1. With the sphere still selected, Copy (Ctrl + C) and Paste (Ctrl + V) or Duplicate (Ctrl + D) the sphere and use the transform tools to set up the remaining 8 planets.

  1. Select one of the sphere in the scene and open the "Configure Tool", under the Model section set the "Material" texture to the "earth.jpg" image in your asset library. Rename the planet accordingly in the configure tool. We will use images of different planets to transform the plain sphere object into objects representing different planets.

  1. Repeat the same procedure for all other spheres.

  1. For each planet, we set their scale relative to the Earth using data from NASA. Open the "Configure Tool" to modify the "Scale" of the planets and use the transform tools to reposition them.

Make the Solar System Orbit and Rotate

  1. Select the Sun object and click on the "Behaviour" tab to switch to its behaviour panel. As gravity keeps all planets orbit around the sun, we will use the Sun object to store the planets' orbital and rotational speed data.

  1. Select the "Show Item Variables" button to open the item variables panel. Add a new variable named "MecuryOrbitalPeriod", set the variable type to "Number" and its default value to 0.241.

  1. Repeat the same procedure to add the orbit and rotation variables data for the other planets. The Sun's variable panel should look similar to the list below.

  1. Create 2 more variables "OrbitSpeedControl" and "RotateSpeedControl" to control the general speed of orbit and rotation. You may adjust the value of these two speed control variables to make all plants orbit or rotate faster/slower.

Set the Mercury to Rotate​

  1. Select the Mercury object and click on the "Behaviour" tab to switch to its behaviour panel.

  1. In the "General Event" category, drag and drop the "Start" event trigger block onto the behaviour canvas. Then switch to the "Flow Control" category, drag and drop the "Repeat" block onto the behaviour canvas, and add the "Rotate" block from "Motion" category to the behaviour canvas as well. Connect the "Start" block's out flow node with the "Repeat" block's in flow node, and "Repeat" block's out flow node with "Rotate" block's in flow node. This sets up the basic logic for the Mercury object to repeatedly rotate when the scene starts.

  1. We then specify how much the object will rotate. In the "Math" category, drag and drop the "Vector3" block onto the behaviour canvas. Then, set the Y value of the "Vector3" block to 360 and connect its Output node with the "Rotate" block's Angle node. We set the planets to rotate 360 degrees repeatedly, yet the rotation will not happen with the Time set to 0.

  2. Drag and drop a "Multiply" block from the "Math" category, then switch to the "Variable" category, drag and drop two "Get Number" blocks onto the behaviour canvas. For both "Get Number" blocks, select "sun" as the "Target", then set one "Key" to "MercuryRotationPeriod" and the other to "RotateSpeedControl". Connect one "Get Number" block's Output node with the "Multiply" block's A node and connect the other "Get Number" block's Output node with "Multiply" block's B node. Finally, connect "Multiply" block's Output node with "Rotate" block's Time node.

  3. This is the close-up of the whole behaviour flow of the Mercury. This sets the Mercury object to rotate 360 degrees against its centre of origin every period, which value is defined as the product of the "MercuryRotationPeriod" and the "RotateSpeedControl" values.

  1. Press the "Play" button to enter play mode, move or teleport to near the Mercury planet and check if it's rotating as expected. Then use the "Esc" button to return to scene editing.

Set the Mercury to Orbit​

  1. We will use an empty object Linked to a Planet to create the Orbit effect. First, add an "Empty Object" to the scene and make sure it's spawned at the origin (same position as the Sun).

  2. Select the "Empty Object" and rename it as mercuryEmpty in the "Configure Tool".

  3. Hide the "Sun" object in editor.

  4. Select the "Mercury" object, press the "Link to another item" button then point and click the mercuryEmpty object to make mercuryEmpty a parent of the Mercury object. Now the Mercury object will move together with the mercuryEmpty parent object while keeping the same relative position / rotation between them.

  5. To confirm the two objects are linked successfully, you can select the "mercuryEmpty" parent object in the scene and observe that the "Mercury" child object is also selected.

  6. With the mercuryEmpty object still selected, click on the "Behaviour" tab to switch to its behaviour panel.

  1. Drag and drop a "Start" event trigger block, a "Repeat" block and a "Rotate" block onto the behaviour canvas in the same way as you did for the Mercury object, then connect the blocks sequentially.

  2. Following the same logic for rotating the "Mercury" object, drag and drop a "Vector3" block and a "Multiply" blocks, then add two "Get Number" blocks onto the behaviour canvas. Set the Y value of the "Vector3" block to 360 and connect its Output node with the "Rotate" block's Angle node. For both "Get Number" blocks, select "sun" as the "Target", then set one "Key" to "MercuryOrbitalPeriod" and the other to "OrbitSpeedControl". Connect one "Get Number" block's Output node with the "Multiply" block's A node and connect the other "Get Number" block's Output node with "Multiply" block's B node. Finally, connect "Multiply" block's Output node with "Rotate" block's Time node.

  3. This is the close-up of the whole behaviour flow of the mercuryEmpty. This sets the mercuryEmpty object to rotate 360 degrees against its centre of origin every period, which value is defined as the product of the "MercuryOrbitPeriod" and the "OrbitSpeedControl" values.

  1. Press the "Play" button to enter play mode, the Mercury planet should be rotating and orbit around the Sun simultaneously. Then use the "Esc" button to return to scene editing.

Set up Orbit and Rotation for Other Planets​

  1. We can apply the same rotate and orbit logic to other planets. Here we will use the Venus as an example.

  2. To start, first Copy (Ctrl + C) and Paste (Ctrl + V) or Duplicate (Ctrl + D) the mercuryEmpty object, make sure it's resided at the origin at (0, 0, 0). Rename the duplicated object as "venusEmpty" and set it as parent object to Venus using the "Link" tool.

  3. Open the Mecury object's behaviour panel then select and copy (Ctrl + C) the whole behaviour flow in the Main layer, which is used to rotate the planet.

  4. Then open the Venus object's behaviour panel, Paste (Ctrl + V) the copied behaviour flow and update the corresponding "Key" in the "Get Number" block attached to Multiply block's A node from "MercuryRotationPeriod" to "VenusRotationPeriod".

  5. Open the mercuryEmpty object's behaviour panel and copy (Ctrl + C) the whole behaviour flow of mercuryEmpty in the Main layer.

  6. Paste the copied behaviour flow to venusEmpty object's behaviour panel, and update the corresponding "Key" in the "Get Number" block accordingly.

  7. Repeat the same procedure for all other planets, except for the Moon.

  8. Press the "Play" button to enter play mode, all planets should be rotating and orbiting at their own speed except for the Moon, which is the Earth's satellite.

Set Moon to Orbit Around Earth​

  1. As the Moon is following the Earth's Orbit around the Sun as well as the Earth's own rotation at the same time, we will simulate this movement using both the earthEmpty parent object and with the Offset Tool.

  2. Select the "Moon" object and use the "Link" button to parent it with the earthEmpty object.

  3. Then copy the X, Y, Z position as displayed in the Earth's Configure Tool and set the Moon's X, Y, Z position to these values. Now the Moon should be completely inside the Earth and sharing the same position.

  4. With the "Moon" object still selected, use the "Offset Tool" or set the Moon's X-Axis Offset value to 2 in the "Configure Tool". Now the Sphere representing the Moon should be shown outside of the Earth object, but its centre still overlaps with the Earth object.

  5. Now we will add behaviour to make the Moon orbit around the Earth. Select the Mercury object's behaviour panel and copy the behaviour defined in the Main layer for making the Mercury rotate. Switch to the Moon's behaviour panel and paste the behaviour flow to the canvas. In the "Get Number" blocks, change the Key to "MoonOrbitalPeriod" and "OrbitSpeedControl" respectively. This time, since we have offset the Moon sphere model in the previous step, rotating the Moon object will result in an orbiting movement of the Moon sphere model.

  6. Enter play mode to check out how the Moon orbits around the Earth.

Create New Planet Scene

Add the Welcome Text​

  1. Open the "New Item Panel" and from the "Specialize Objects" tab, select and add the "Text Object" to the scene to create a Welcome Textbox.

  2. Use the transform tools (move / rotate / scale) to reposition the text object. Then open the "Configure Tool" to set up the text content, font size, and colour of the text object.

Set up the Mercury Scene​

  1. Open the "Scene Menu" and select the "Add New Scene" button to create a new scene.

  2. Select the "Empty Scene" template and add it to the project.

  3. Rename the Empty Scene as "Mercury Scene" and press "Change to" to open the new Mercury Scene.

  4. In the Mercury Scene, open the "New Item Panel" and add a "Ground" object to the scene.

  5. Modifying the "Scale" of the Ground object using either the transform tools or directly in the "Configure Tool". We want to have a large enough area for the player to move around.

  6. Add a "2D Image Object" to the scene. We will be using this 2D Image Object to simulate the appearance of Mercury in the scene.

  7. Open the "Configure Tool" and modify the "Scale" of the 2D Image Object so it's in similar size as the Ground object. Then, click the selector next to the "Image Asset" option to open the Selection panel and select the "mercury.jpg" image in your asset library to simulate mercury's ground appearance. This is the same image used for the Sphere of Mercury planet, but shown in 2D format instead.

  8. Click at any empty space in the scene to select the Scene and open its "Configure Tool". Here we use the same "stars_milky.jpg" in your asset library to set the skybox 360 image.

  9. Open the "New Item Panel" and from the "Imported Assets" tab, select and add the 3 "rock" objects to the scene. Use the transform tools (move / rotate / scale) to adjust the size and position of the rock objects.

  10. Select one of the rock object and open its "Configure Tool". Make sure both the "Interactable" and "Grabbable" options are selected. Then, "Enable" the Physics setting as well as the "Use Gravity" option . Repeat these procedures for the other rock objects.

  1. You have now finished setting up the grab interaction and enabled gravity for the rock objects. Press the "Play" button and press the right mouse button to grab the rocks to test. They should fall to the ground with simulated Earth gravity when you release the mouse button.

  2. Press "Esc" to exit play mode after you've done testing your scene.

Use Add Force to Simulate Gravity of the Planet​

  1. Since the "Use Gravity" option simulates the Earth gravity, we want to turn off the earth gravity and simulate the Mercury gravity using Physics behaviour blocks. Before setting the behaviour flow, first open the rocks' Configure Tool and deselect "Use Gravity" under Physics setting while leaving Physics "Enabled".

  2. Select a rock object and click on the "Behaviour" tab to switch to its behaviour panel.

  1. Drag and drop a "Start" event trigger block, then a "Repeat" block and "Wait For" block from the "Flow Control" category to the behaviour canvas. Switch to the "Physics" category, drag and drop the "Add Force" block onto the behaviour canvas. Connect the blocks sequentially. Then set the Time on "Wait For" block to 0.1. We want to add force to the rock object every 0.1 second.

  2. From the "Math" category, drag and drop a "Vector3" block to the canvas, then connect its Output node to the "Add Force" block's Force node. Set the Y value of "Vector3" to -0.37. (The equation of the Force Add in Mercury = -(Gravity of Mercury 3.7 divided by 10 FPS)). We fix the FPS to 10 frames per seconds. So every 0.1 seconds, we add one tenth of the Mercury's gravity to the rock in a downward direction.

  3. Select the whole behaviour flow, Copy (Ctrl + C) and Paste (Ctrl + V) the behaviour flow to the other two rock objects.

Add Welcome Text and Return Button when Clicked​

  1. Open the "New Item Panel" and from the "Specialized Objects" tab, select and add the "Text Object" to the scene.

  2. Use the transform tools (move / rotate / scale) to reposition the text object. Then open the "Configure Tool" to set up the text content, font size, and colour of the text object.

  3. Open the "New Item Panel" and from the "Specialized Objects" tab, select and add the "Text Object" to the scene.

  4. Use the transform tools (move / rotate / scale) to reposition the button object. Then open the "Configure Tool" to set up the text content, font size, and colour of the button object.

  5. Select the button object and open its behaviour panel. Under "Groups", add a new behaviour group named "return" and toggle the "Action" button to make it an action that can be called by other objects. An "Action Start" event trigger block will appear on the canvas.

  6. Switch to behaviour Library, under the "Scene" category drag the "Change Scene" block and append it behind the "Action Start" block. Select the "Solar System" scene under "Change Scene" block's "Scene" option, then connect the "Action Start" block and the "Change Scene" block. When this action is called, the scene will be changed back to the Solar System.

  7. Select the "Button Object" and open its "Configure Tool" scroll down to select the "Action" option for the button. On the selection panel, select the button object from the item list on the left side, then choose the "return" action on the right and select the action. This will set up the "Click here to return solar system" button object to call the "return" action to change back to the "Solar System" scene when the button is clicked.

Add Outline Effect to Objects​

  1. In the Mercury scene, open the behaviour panel for one of the rocks. Construct a behaviour flow using "Start" event trigger block, "Color" Math block and "Outline" Effect block. Set the "R" (red) value of the Color block to 255 and the "A" (alpha) value of the Color block to 255, and all other values to 0, the color preview under the "Output" node should show a red color. Connect all the blocks in the same manner as shown below. This gives the rocks a red outline when scene starts.

  2. Drag the behaviour canvas to an empty space, create another behaviour flow using "Start Pointing" event trigger block, a "Color" Math block and an "Outline" Effect block. Set the "G" (green) value of the Color block to 255 and the "A" (alpha) value of the Color block to 255, and all other values to 0, the color preview under the "Output" node should show a green color. Connect all the blocks in the same manner as shown below. This gives the rocks a green outline when pointed at with the Wand.

  3. Copy and paste the above behaviour blocks to another empty space on the canvas, then replace the "Start Pointing" block with a "Stop Pointing" block. Set the "Color" block value to red for this behaviour flow. This time, the rocks will again show a red outline when the Wand stops pointing at it.

  4. Select the whole outline effects behaviour flow, Copy (Ctrl + C) and Paste (Ctrl + V) all the behaviour flows to the other two rock objects.

  5. Deselect any objects in the scene and open the "Configure Tool" to configure the scene settings. Select "Wand" as the "Default Tool". This sets the Wand tool as the default interaction tool in XR mode. Then change the "Wand Length" into "10".

  6. You have now finished setting up the outline effects for the rock objects. Press the "Play" button and test your scene. The rocks would show a green outline when pointed and a red outline in other circumstances. Press "Esc" to exit play mode after you've done testing your scene.

  7. We can apply the same technique to add outline to the planets in the "Solar System" scene. Use the Mercury object as an example, we first select it and open its behaviour panel.

  8. Paste (Ctrl + V) the whole outline effects behaviour flow onto the Mercury object's Main layer canvas.

  9. We also add another behaviour flow to the Mercury object, so the user will be directed to the Mercury scene when the Mercury object is clicked.

  10. You have now finished setting up the outline effects for the Mercury planet. Press the "Play" button and test your scene, then use the Wand to click the "Mercury" planet to enter the Mercury scene.

  11. Press "Esc" to exit play mode after you've done testing the change scene feature.

Set up other Planets' Scenes

  1. You can use the Mercury scene as a template to create and set up scenes for the other planets. This can be achieved using the "Duplicate Scene" button on the Scene panel.

  2. From any scene, open the "Scene Menu" and press the "Duplicate" button for the Mercury scene to duplicate the scene and all objects configured in the scene. Rename the scene to Venus then double click the Venus scene to enter.

  3. Select the text object in the Venus scene and edit its Text content using the "Configure Tool", change the planet name from Mercury to Venus.

  4. Then replace the "Image Asset" that's used as ground appearance to "venus.jpg".

  5. Lastly, select the rocks on the ground and open their behaviour panel.

  6. Change the add force data in the "Vector3" block to -0.89 to simulate Venus gravity. (The equation of the Force Add in Venus = -(Gravity of Venus/FPS))

  7. Repeat the same procedure for other planets.
